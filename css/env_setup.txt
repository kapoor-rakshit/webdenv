CSS is Render Blocking
REFERENCE: https://web.dev/critical-rendering-path-render-blocking-css/
By default, CSS is render blocking resource, which means that browser won't render DOM until CSSOM (CSS Object Model) is constructed.
Media types and Media queries allow to mark some CSS resources as non-render blocking.
Only if media query/type conditions match, browser blocks rendering until style sheet is downloaded and processed.
Although, browser downloads all CSS resources, regardless of blocking or non-blocking behavior, albeit with lower priority for non-blocking resources.
Eg:
<link href="style.css" rel="stylesheet" />                                  // render blocking | no media query/type
<link href="style.css" rel="stylesheet" media="all" />                      // render blocking | media="all" (default)
<link href="portrait.css" rel="stylesheet" media="orientation:portrait" />  // non-blocking for landscape | Depend on orientation of device while page is loading
<link href="print.css" rel="stylesheet" media="print" />                    // non-blocking for media types (screen, speech) other than print
<link href="other.css" rel="stylesheet" media="(min-width: 40em)" />        // non-blocking for width less than 40em

BEM (Block, Element, Modifier) methodology
--------------------------------------------
It is a naming convention for classes in HTML and CSS.
In this CSS methodology a BLOCK is a top-level abstraction of a new component, for example a button: .btn { }. This block should be thought of as a parent.
Child items, or ELEMENTs, can be placed inside and these are denoted by two underscores following the name of the block like .btn__price { }.
MODIFIERs can manipulate block so that we can theme or style that particular component. This is done by appending two hyphens to name of block like btn--orange.
Eg:
CSS
  .btn {}           /* BLOCK component */
  .btn__price {}    /* ELEMENT that depends upon block */ 
  .btn--orange {}   /* MODIFIER that changes style of block */
HTML
  <a class="btn btn--big btn--orange" href="https://css-tricks.com">
    <span class="btn__price">$9.99</span>
    <span class="btn__text">Subscribe</span>
  </a>

CSS VARIABLES
---------------
- A custom property name / variable begins with a double hyphen (--)
- NOTE: Selector given to the ruleset defines the scope that custom property can be used in.
        A common best practice is to define custom properties on the :root pseudo-class, so that it can be applied globally across your HTML document
        Custom properties do inherit. This means that if no value is set for a custom property on a given element, the value of its parent is used
- Custom property names are case sensitive i.e. --my-color will be treated as a separate custom property to --My-color
- Use the custom property value by specifying custom property name inside the var() function
- Using the second arg of var() function, we can define fallback value
- element.style.getPropertyValue("--my-var");              // JS : get variable from inline style
  getComputedStyle(element).getPropertyValue("--my-var");  // JS : get variable from wherever
  element.style.setProperty("--my-var", jsVar + 4);        // JS : set variable on inline style
eg:  :root {
       --main-bg-color: brown;
     }
     .one {
       background-color: var(--main-bg-color);
       color: var(--my-var, red);
       text-decoration: var(--my-var, var(--decor-var, none));
     }

                                            Lightness is also a percentage; 0% is dark (black) and 100% is white.
5. transparent
   makes a color transparent. The transparent keyword is equivalent to rgba(0,0,0,0) i.e. opacity or alpha is 0
   
6. currentcolor
   It is like variable that holds current value of `color` property of element. example: body{ color: purple;}  div{ background-color: currentcolor;}

/* CSS Grid Layout */
/* An HTML element becomes a grid container when its display property is set to grid or inline-grid. */
/* All direct children of the grid container automatically become grid items. eg: <div class="grid-container"><div class="grid-item">1</div></div> */
.grid-container {
  display: grid;
  grid-template-columns: auto auto auto;     /* number of columns in your grid layout, and define the width of each column */
  grid-template-rows: auto auto;             /* height of each row */
  grid-column-gap: 50px;                     /* gap between columns */
  grid-row-gap: 50px;                        /* gap between rows */
}
.grid-item {
  grid-column-start: 1;                      /* just like colspan in <table> or Merge cells in excel , start at column line number 1 */
  grid-column-end: 3;                        /* end at column line number 3 */ /* NOTE : It is column line number not column number */
  grid-row-start: 1;                         /* similar to colspan but merge row wise , start at row line number 1 */
  grid-row-end: 3;                           /* end at row line number 3 */ /* NOTE : It is row line number not row number */
}

/* CSS Flexbox (Flexible Box) Layout */
/* The Flexible Box Layout Module, makes it easier to design flexible responsive layout structure without using float or positioning. */
/* Parent flex container becomes flexible by setting display property to flex *//* eg: <div class="flex-container"><div class="item">1</div><div>2</div></div> */
.flex-container {
  display: flex;
  flex-direction: row;            /* which direction (main-axis) container wants to stack flex items, column(y-axis) | column-reverse | row(default)(x-axis) | row-reverse */
  flex-wrap: wrap;                /* whether flex items should wrap or not, wrap | nowrap (default) | wrap-reverse */
  flex-flow: row wrap;            /* shorthand property for setting both the flex-direction and flex-wrap properties */
  justify-content: center;        /* align flex lines (containing items)(along main-axis), center | flex-start(default) | flex-end | space-evenly | space-between */
  align-content: center;          /* align flex lines (containing items)(along cross-axis), stretch (default) | space-around | center | flex-start | flex-end */
                                  /* IMP NOTE: align-content has no effect on single line flex containers or ones with flex-wrap: nowrap */
  align-items: center;            /* align flex items (within flex line)(along cross-axis), stretch (default) | center | flex-start | flex-end | baseline */
}
.flex-container > .item {
  background-color: #f1f1f1;
  margin: 10px;
  order: 0;                       /* sequence of items i.e. HTML code and rendered page can have different sequence of items, (default: 0) */
  flex-grow: 0;                   /* how much flex item will expand on main-axis relative to rest of flex items, (default: 0) */
  flex-shrink: 1;                 /* how much flex item will shrink on main-axis relative to rest of flex items, 0 value will not let item shrink, (default: 1) */
  flex-basis: 200px;              /* specifies the initial length of a flex item */
  align-self: flex-end;           /* overrides default alignment of that item set by parent container's 'align-items' property */
  flex: 0 1 200px;                /* shorthand property for the flex-grow, flex-shrink, flex-basis properties */
}

CSS SPECIFICITY
--------------------
If there are two or more conflicting CSS rules that point to same element, browser follows some rules to determine which one is most specific and therefore wins out.
Specificity is a score/rank that determines which style declarations are ultimately applied to an element.
Four categories which define the specificity level of a selector (start at 0):
- Inline styles                             : add 1000 for style attribute
- IDs                                       : add 100 for each ID
- Classes, attributes and pseudo-classes    : add 10 for each attribute, class or pseudo-class
- Elements and pseudo-elements              : add 1 for each element name or pseudo-element
- The universal selector(*) and inherited values have a specificity of 0
